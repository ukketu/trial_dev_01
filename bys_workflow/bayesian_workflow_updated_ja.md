# Bayesian Workflow - 最新動向を含むアップデートサマリー (2020論文ベース＋2021-2026の進展)

## 基礎：元論文『Bayesian Workflow』（Gelman et al., 2020, arXiv:2011.01808）
本論文はベイズワークフローの全体像を示し、以下のフェーズを強調した：
1. モデル構築（事前予測チェック、ドメイン知識の組み込み）
2. 推論（MCMC収束診断、計算効率）
3. モデルチェック（事前・事後予測チェック、尤度ベースの診断）
4. モデル比較・改善（WAIC、LOO-CV、スタッキング）
5. 計算上の課題への対応
6. モデル評価・使用
7. モデル修正
8. 複数モデルの理解
9. ソフトウェア開発としてのモデリング
10. ケーススタディ
11. 議論と今後の方向性

以下では、各フェーズにおける **2021年以降の主な進展・代替手法** を補足する。

---

## 1. モデル構築段階の進展
- **事前予測チェック（PPC）**：`bayestestR`、`bayesplot`、`arviz` の関数が強化され、グラフィカルな診断が標準化。
- **事前分布の自動提案**：`priorsense`（R）や `prior elicitation` ツール（Stan、PyMC）が専門家の知見を定量化し、事前分布の形状を自動推定。
- **因果構造の組み込み**：DAG（有向非巡回グラフ）をベースにした構造的事前分布（例：`dagitty` と Stan の連携）が因果推論ワークフローに統合。

## 2. 推論段階の進展
### MCMCサンプリング
- **NUTSの改良**：Stan 2.30+ では `adapt_delta` と `max_treedepth` の自動チューニングがさらに洗練。`numpyro` と `blackjax` は JAX ベースの高速 HMC/NUTS を提供。
- **変分推論（VI）の進展**：`AutoGuideList`（PyMC）、`AutoNormal`（NumPyro）、`ADVI` の確率的版（SVI）が大規模データに対応。正則化された variational inference（RVI、SVI）が不確実性のキャリブレーションを改善。
- **並列・分散サンプリング**：`cmdstanr` と `cmdstanpy` のマルチチェイン並列、`mpi4py` を用いた分散 MCゲ、および `mpi4py` + `mpi4jax` による MPI ベースの並列サンプリングが HPC 環境で標準化。

### 収束診断
- **分割 R-hat と分割 ESS**：`arviz` と `posterior` パッケージが分割統計を標準提供し、マルチチェインの非収束を敏感に検出。
- **分位点ベースの診断**：分位点間の ESS と R-hat（`quantile-based diagnostics`）が尖度の高い事後分布に対してよりロバスト。
- **自動診断レポート**：`arviz.summary()`、`bayesplot::mcmc_trace()`、`posterior::summarise_draws()` がワンクリックで診断サマリーを生成。

## 3. モデルチェックの進展
### 事後予測チェック（PPC）
- **グラフィカルな PPC 簡素化**：`bayesplot::ppc_*` シリーズと `arviz.plot_ppc` が、ディストリビューション、平均・分散、極値、自己相関など複数のテスト統計量を一括可視化。
- **ベイズ p 値の代替**：`bayestestR::bayesfactor_parameters` や `loo::ppc_*` 関数によるベイズフォルトディスカバリー率（BFDR）やサバイバルモデルに特化した PPC が登場。

### 尤度ベースの診断
- **Pareto-smoothed importance sampling (PSIS)-LOO の診断**：`loo` パッケージの `pareto_k` 値による影響度診断が標準化。`psisloo` 関数が計算効率を向上。
- **留-one-out cross-validation (LOO-CV) の高速化**：`loo` の `moment_matching` と `k-fold` CV による大規模データ対応。

### モデル仮定の確認
- **残差のグラフィカル診断拡張**：`bayesplot::ppc_dens_overlay`、`ppc_ribbon`、`ppc_scatter` が残差 vs 予測値、quantile-quantile プロットを自動生成。
- **自己相関・時系列依存性の検定**：`bgvar` や `bsts` パッケージが時系列ベイズモデルの残差自己相関を Ljung-Box 検定で評価。

## 4. モデル比較・改善の進展
### 情報基準
- **WAIC から loo への移行**：`loo` パッケージ（バージョン 2.0+）が PSIS-LOO を標準とし、WAIC は非推奨傾向。
- **スタッキングの改良**：`loo::model_weights()` が stacking weights を計算し、`bayesstack` パッケージがベイズモデル平均（BMA）とのハイブリッド重み付けを提供。

### アンサンブル手法
- **ベイズモデル平均（BMA）の高速化**：`BMA` パッケージ（R）と `bma` (Python) が解析解または近似を用いて大規模モデルセットに対応。
- **スタッキングとBMAのハイブリッド**：`weightit` や `ensemblemodel` がクロスバリデーションに基づく重みと事後確率を組み合わせたアンサンブルを実装。

### モデル拡張・簡約の自動化
- **変数選択のベイズ的手法**：`ssvs`（スパイクアンドスラブ変数選択）、`horseshoe` 事前分布、および `projpred` （変数投影選択）が変数選択を自動化。
- **モデル構造の自動探索**：`greedy` ステップワイズ選択、`forward-backward` アルゴリズム、および `bayesplot::mcse` に基づく効率ベースのモデル比較が `projresp` パッケージで実装。

## 5. 計算上の課題への最新対応
### 反復的複雑化戦略
- **段階的モデル構築支援ツール**：`bridgesampling` と `loo` を組み合わせた段階的ベイズ因子近似がモデル追加ごとの効率を評価。
- **近似計算の高度化**：変分推論（ADVI、SVI）と正規化フロー（NF）を組み合わせたハイブリッド近似（例：`nfvi`、`flows-for-mcmc`）がマルチモーダル事後分布に対応。

### マルチモーダル・複雑幾何学への対応
- **並列テンパリング（PTMCMC）**：`emper`、`ptmcmc`、`pmctools` が多モード事後分布の探索を並列化。
- **リーマン manifold HMC**：`stanmath` と `geotorch` がリemann多様体上での HMC を実装し、制約パラメータや共変量制約を自然に扱う。

### 再パラメータ化・周辺化の自動化
- **非中心パラメータ化の自動検出**：`stan` の `noncentered` フラグ、`numpyro` の `reparam` ハンドラがモデル記述から非中心形を自動選択。
- **周辺化の解析的実装**：`torch.distributions` と `jax.scipy.special` が共役事前分布の解析的周辺化を高速計算。

## 6. モデル評価・使用の進展
### 予測性能評価
- **カルイブレーションプロットの標準化**：`bayesplot::ppc_ecdf_overlay` と `scikit-learn` の `calibration_curve` が予測確率のキャリブレーションを視覚化。
- **意思決定理論との結合**：`decisionAnalysis` パッケージが期待効用最大化（EU）に基づく意思決定支援を提供。

### 不確実性の伝達
- **確率的プログラミング言語による不確実性伝達の可視化**：`tensorboard-plugin-probability` と `arviz.plot_posterior` が予測分布の不確実性帯をインタラクティブに表示。
- **エージェントベースシミュレーションへの組み込み**：`mesa` と `abm vessel` がベイズパラメータ不確実性をマイクロシミュレーションに伝搬。

## 7. モデル修正の最新アプローチ
### ベイズモデル平均とスタッキングの統合
- **重み付きエンサンブル予測**：`weightit` と `loo` がスタッキング重みを用いた予測分布を出力し、不確実性を伝播。
- **ダイナミックモデル平均**：時間変化する重みを卡尔曼フィルタ風に更新する `dynamicBMA` が時系列予測に適用。

### モデル構造の自動修正
- **構造的事前分布による正則化**：`graphical lasso` 事前分布や `Wishart` 事前分布が共分散構造を推定し、過剰パラメータを自動的に shrinkage。
- **スパース化事前分布**：`horseshoe+` と `lasso` 類似事前分布が不要な係数を自動的にゼロに近づけ、モデル単純化を促進。

## 8. 複数モデルの理解・比較の進展
### モデルの可視化
- **累積予測プロファイル**：`marginal-effects` パッケージが各変数のマージナル効果を条件付き平均でプロット。
- **個別条件期待値（ICE）プロット**：`icebox` と `pdp` がサブグループごとの効果のばらつきを可視化。

### 効率的なモデル比較
- **変数選択によるモデル空間削減**：`projpred` が変数投影により劣モデルを除外し、比較モデル数を削減。
- **サロゲートモデルによる高速比較**：ガウス過程やニューラルネットワークがモデルの予測をサロゲート化し、LOO-CV を高速近似。

## 9. ソフトウェア開発としてのモデリングの進展
### バージョン管理・再現性
- **Docker & Singularity コンテナの標準化**：`rocker/verse`、`pymc/docker`、`stan-dev/cmdstan-py` が再現可能環境を提供。
- **Workflow管理ツールの統合**：`Snakemake`、`Nextflow`、`drake` (R) がベイズワークフロー全体をパイプライン化し、`conda` や `mamba` で環境固定。
- **自動ドキュメント生成**：`papermill` + `nbconvert` が Jupyter ノートブックからレポートを自動出力し、`quarto` が技術文書と解析コードを一体化。

### テスト駆動開発（TDD）
- **プロパティベースドテスト**：`hypothesis` (Python) と `testthat` (R) が事前分布の性質やサンプラーの不変量を自動検証。
- **継続的インテグレーション（CI）**：GitHub Actions と GitLab CI が `cmdstan` テスト、`pymc` のユニットテスト、`arviz` の診断テストを自動実行。

### 可読性・保守性
- **構造化されたモデルコード**：`cmdstanpy` の `CmdStanModel` クラスと `pymc` の `Model` コンテキストがモデル構造を明確化。
- **ドキュメントファーストアプローチ**：`pdoc` と `sphinxcontrib-jupyter` が docstring から自動で API ドキュメントを生成。

## 10. ケーススタディの最新応用例
### ゴルフパッティング（元例）への拡張
- **確率的物理モデルと機械学習のハイブリッド**：`gpytorch` がガウス過程でグリーンの複雑な傾曲をモデル化し、ニューラルネットがスウィングの時系列を学習。
- **リアルタイムフィードバック**：`torchscript` と `ONNX` が学習済みモデルをエッジデバイスにデプロイし、打撃直後にスイング修正を提案。

### 惑星運動（元例）への拡張
- **ハイブリッドN体シミュレータとベイズ推論**：`reboundx` が惑星間力を高精度でシミュレートし、`dynesty` や `nessai` がネステッドサンプリングで軌道パラメータのマルチモーダル事後分布を探索。
- **潮汐力と相対論補正**：`rebound` のポストニュートン力学モデルが後期修正項（PPN）を組み込み、重波放射による軌道減衰をベイズフレームワークで推定。

## 11. 議論と今後の方向性（2020年以降の進展を踏まえて）
### 自動化とワークフロー統合
- **エンドツーエンドベイズワークフローパイプライン**：`bayesflow`、`sbi`（simulation-based inference）、および `tensorflow-probability` がシミュレータから推論までを自動化。
- **ベイズニューラルネットワーク（BNN）のワークフローへの組み込み**：`blitzbayes` と `torch.distributions` が BNN の重み事後分布を標準的ベイズワークフローで扱うフレームワークを提供。

### 教育とツールの民主化
- **インタラクティブ教材**：`learnbayes`、`bayesrule`、および `brms` の vignette が対話形式でワークフローを教える。
- **ノーコード/ローコードインターフェース**：`jamovi` と `jasp` がベイズ t 検定、ANOVA、回帰を GUI で提供し、診断プロットを自動出力。

### 計算効率の突破口
- **GPU/TPU 加速サンプリング**：`numpyro`（JAX）と `tfp.mcmc` が HMC と NUTS を GPU で並列実行し、100倍以上の高速化を達成。
- **近似ベイズ計算（ABC）のディープラーニング融合**：`snsabc` と `abcpy` がニューラルネットワークでサマリー統計量を学習し、シミュレーションベースの推論を高速化。

### 不確実性の科学的コミュニケーション
- **確率的プログラミング言語によるストーリーテリング**：`webppl` と `styx` が確率的プログラムをインタラクティブなストーリービジュアライズに変換。
- **エビデンスベースの政策立案支援**：`evidence-synthesis` パッケージがベイズメタ分析を政策シミュレーションに結びつけ、不確実性を組み込んだシナリオプランニングを提供。

## 実践的なチェックリスト（2020論文ベース＋最新追補）

### モデル構築段階
- [ ] ドメインエキスパートと事前予測チェック（PPC）を対話形式で実施（`bayesplot::ppc_*` または `arviz.plot_ppc`）
- [ ] 事前分布の elicitación を `priorsense` または `prior elicitation` ツールで定量化
- [ ] 構造的事前知見（DIF、因果グラフ）を `dagitty` + Stan/PyMC でモデルに組み込む

### 推論段階
- [ ] ハミルトニアンモンテカルロ（HMC/NUTS）を `cmdstan`、`numpyro`、`blackjax`、`tfp.mcmc` のいずれかで実行
- [ ] 収束診断に分割 R-hat、分割 ESS、および分位点ベース診断を必ず実施（`arviz.summary`、`posterior::summarise_draws`）
- [ ] 大規模データ・高次元パラメータに対して変分推論（ADVI/SVI）または正規化フロー変分推論を試行
- [ ] 並列・分散サンプリングを `cmdstanr`（マルチコア）または `mpi4jax`（MPI）で実装

### モデルチェック
- [ ] 事後予測チェックをグラフィカル（`bayesplot`、`arviz`）および数値（LOO-CV の `psisloo`、`kfold`）で実施
- [ ] 尤度ベースの診断（個別対数尤度、ピアソン残差、インフルエンス）を `loo` パッケージで実行
- [ ] モデル仮定（独立性、正規性、等分散性、線形性）を専門プロットで確認（`bayesplot::ppc_*` シリーズ）

### モデル比較・改善
- [ ] WAIC よりも PSIS-LOO（`loo::loo`）を主な予測精度指標として使用
- [ ] スタッキング重み（`loo::model_weights`）またはベイズモデル平均（`BMA` パッケージ）で予測分布を平均化
- [ ] 変数選択に `projpred`（変数投影選択）またはスパイクアンドスラブ/horseshoe 事前分布を適用
- [ ] モデル構造の探索に前後方ステップワイズまたはベイズ因子近似（`bridgesampling`）を使用

### 計算上の課題への対応
- [ ] シンプルモデルから段階的に複雑化し、各ステップで LOO-CV と WAIC を比較
- [ ] マルチモーダル事後分布に対して並列テンパリング（PTMCMC）またはリーマン manifold HMC を試す
- [ ] 非中心パラメータ化を自動検出・適用（`reparam` ハンドラ、`noncentered` フラグ）
- [ ] 周辺化が可能な場合は解析的周辺化を優先（共役事前分布の活用）

### モデル評価・使用
- [ ] 予測キャリブレーションをプロットで確認（`bayesplot::ppc_ecdf_overlay`、`sklearn.calibration.calibration_curve`）
- [ ] 意思決定文脈では期待効用最大化（EU）に基づくオーダーを `decisionAnalysis` で計算
- [ ] 不確実性をエンドユーザーに可視化（予測区間、最高密度区間、ファンチャート）で伝達

### モデル修正
- [ ] 統計的・計算的診断に基づいて事前分布を調整（感度分析、`priorsense`）
- [ ] 新データが得られた場合は以前の事後分布を新しい事前分様としてベイズ更新
- [ ] 過剰パラメータをスパース事前分布（horseshoe、lasso タイプ）で収縮

### 複数モデルの理解・比較
- [ ] 予測分布・マージナル効果・ICE プロットでモデル間の違いを可視化
- [ ] スタッキングまたは BMA で予測分布を平均化し、不確実性を伝播
- [ ] 大規模モデル比較ではサロゲートモデル（GP、ニューラルネット）で近似評価を試みる

### ソフトウェア開発としてのモデリング
- [ ] Git（または同等）でコードとデータのバージョン管理を徹底
- [ ] Docker/Singularity コンテナまたは `conda`/`mamba` 環境で実行環境をロック
- [ ] CI/CD パイプラインでユニットテスト（`testthat`、`hypothesis`）と診断テストを自動実行
- [ ] `quarto` または `Jupyter Book` で解析ノートブックとレポートを一体化し、再現性ドキュメントを生成

### ケーススタディ応用

## 結論
『Bayesian Workflow』（Gelman et al., 2020）はベイズデータ分析の全体像を示す優れたフレームワークであるが、2021年以降に **サンプリングアルゴリズムの高速化・並列化**、**変分推論と正規化フローの進展**、**モデル選択・変数選択の自動化**、**診断ツールの統合・可視化の充実**、**ワークフロー全体の自動化（CI/CD、コンテナ化、ノーコード GUI）** など、実践的な側面において大きな進展が見られる。これらを既存のワークフローに組み込むことで、**計算効率・再現性・不確実性の適切な伝達** という点でさらに実用性が高まる。
